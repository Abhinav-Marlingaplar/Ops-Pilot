How it works

Five components.
One pipeline.

A clean separation of concerns — each component does exactly one thing and does it well.

01 / TRIGGER
Webhook
POST /webhook with repo, branch, commit. JWT-authenticated. Build row inserted into Postgres instantly.
→
02 / QUEUE
RabbitMQ
Durable build_jobs queue decouples job intake from execution. Jobs survive broker restarts.
→
03 / EXECUTE
Worker
Clones repo, runs npm install + npm test, builds Docker image. Streams every log line live.
→
04 / STORE
PostgreSQL
ACID-compliant build records, full log storage, status history. All queryable.
→
05 / OBSERVE
Dashboard
React + Socket.IO. Live build list, stage tracker, streaming terminal. No refresh needed.
